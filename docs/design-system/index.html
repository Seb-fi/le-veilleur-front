Espace

Échelle d'espacement tokens/spacing.css

Multiples de 4px. Le rythme dominant est --sp-6 (24px) entre éléments, --sp-12 entre sections, --sp-24 en fin de page.

Mouvement

Durées et courbes tokens/motion.css

Toutes les transitions UI restent sous 280ms. Pas de bounce. Pas de spectaculaire. La courbe par défaut est éditoriale : cubic-bezier(.2, .7, .3, 1).

05 · Vue API

Types et composables — l'implémentation.

Toute interaction non triviale est encapsulée dans un composable. Le code Vue reste descriptif et lisible.

Types

Types métier vue/types.ts

Les types sont minces — ils décrivent uniquement ce que l'UI a besoin de connaître.

TypeDescription
PisteUn fil de pensée. Origin: user ou system. Statut: active / dormant / emerging / tension.
ArticleArticle sauvegardé en favoris. Peut être rattaché à 0..n pistes.
NoteRéflexion personnelle. Peut être épinglée comme centerpiece d'une piste.
DomainEventFait externe daté (annonce, régulation, release).
TraceUnion typée : note | article | event. Utilisée par le stream.
GraphEdgeArête entre deux pistes. Kind: user / implicit / system.
DropDecision"move" | "link" | "cancel". Renvoyé par la modal de drop.
PopoverContextUnion qui définit ce que le popover affiche : empty / entity / note / picker.
Composables

Trois composables couvrent toutes les interactions useMemoryGraph · usePopover · useArticleDnD

Tous renvoient une API stable. Tous sont stateless en dehors du return — n'utilisent ni provide/inject ni store global. À composer dans le composant racine du mode Mémoire.

ComposableResponsabilitéAPI clé
useMemoryGraph État du graphe : clusters, expand/collapse, focused piste, création. toggleCluster(id) · collapseAll() · addPisteAt(name, xy) · attachArticleToPiste(id, art)
usePopover Un popover ouvert à la fois. Auto-close au clic extérieur + Esc. Anti-overflow. open({ position, context }) · close() · isOpen
useArticleDnD Drag-drop d'un article vers une autre piste. Renvoie une décision au drop. beginDrag(e, payload) · callback onDrop(decision) 
// composant racine du mode mémoire
import { useMemoryGraph } from "@veilleur/ds/vue/composables/useMemoryGraph"
import { usePopover }     from "@veilleur/ds/vue/composables/usePopover"
import { useArticleDnD }  from "@veilleur/ds/vue/composables/useArticleDnD"

const graph   = useMemoryGraph({ pistes, satellitesByPiste, edges })
const popover = usePopover()
const dnd     = useArticleDnD({
  resolveTargetAt: (xy) => graph.resolveTargetUnder(xy),
  onDrop: (pending) => {
    // open the move/link modal with `pending`
    pendingDrop.value = pending
  }
})
Components

SFC scaffolds — à recopier, pas à dépendre

Les composants Vue livrés ne sont pas une lib runtime — ils sont des scaffolds canoniques documentés à partir desquels chaque équipe construit. Cela évite le couplage et garde la maîtrise locale du visuel.

SFCRôle
ActionPopover.vueRend le popover contextuel selon usePopover().context. Quatre variantes (entité / note / vide / picker).
MoveLinkModal.vueModal de décision sur drop. Émet decide(decision, pending). Copywriting livré tel quel.
MemGraph.vueCanvas SVG du mode Graphe. Compose useMemoryGraph + useArticleDnD + usePopover.
PisteCard.vueItem cliquable dans l'index des pistes (rail gauche).
StreamItem.vueTrace unique (note / article / event) avec annotation utilisateur.
